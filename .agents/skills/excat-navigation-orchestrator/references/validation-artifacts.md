## Rules

- Create the directory if it does not exist.
- **Locale selector (when hasFlags=true):** Country names and flag image references MUST be in nav.plain.html. header.js must only READ that content from the nav DOM and implement behavior (open/close, selection). header.css handles layout and styling (e.g. multi-column grid). Do NOT put country names or flag URLs in header.js — validate-nav-content.js and the hook enforce this.
- **Image validation (validate-nav-content.js):** After nav.plain.html write, run the script. **ALL images MUST use** `<img src="images/filename.ext" alt="...">` in nav.plain.html. **Use relative paths only — no leading /** (e.g. `images/logo.svg`). Absolute paths like `/content/images/logo.svg` resolve to filesystem root and fail validation. The script checks: (1) image refs satisfy phase-2/3 requirements, (2) each local file exists on disk, (3) each file has size > 0 (no broken/empty downloads). On pass, writes `.nav-content-validated` with `{ imageCount, requirementCount, validatedOnDisk }`. Hook blocks until marker exists and script exit 0. nav.md is not supported.
- **Image audit (audit-header-images.js):** After validate-nav-content.js passes, run the image audit. It builds **expected** image count from phase-2 (rows with hasImages), phase-3 (megamenu/columns hasImages), and megamenu-mapping.json (panel items with hasImage, feature cards, featured area). It compares to **actual** images referenced in nav and on disk. If actual < expected, it writes `image-audit-report.json` with `missingByLocation` (e.g. "Megamenu \"Industries\" → Refining Technologies") so you know where the gap is. On pass, writes `.image-audit-passed`. Hook blocks until this marker exists; Stop check also requires image audit passed.
- **Image manifest compare (optional flow):** When using URL mode to produce `source-image-manifest.json` and `migrated-image-manifest.json`, run `audit-header-images.js --compare=source-image-manifest.json --against=migrated-image-manifest.json --validation-dir=migration-work/navigation-validation`. On pass, writes `.image-manifest-compare-passed`. Hook (IMAGE_PARITY) blocks when both manifests exist until this marker exists.
- Write each file **immediately after** producing that phase's JSON; do not proceed to the next phase until the file is written.
- Paths are relative to workspace root (e.g. `migration-work/navigation-validation/phase-1-row-detection.json`). Do not use `/workspace/` prefix.
- If a phase is skipped (e.g. no megamenu), still write the file with the valid schema-shaped JSON (e.g. zero/empty values and notes).
