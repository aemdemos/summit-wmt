# Troubleshooting — Footer Orchestrator

| Issue | Cause | Action |
|-------|--------|--------|
| Sub-agent JSON rejected or incomplete | Output does not match schema | Re-run sub-agent with schema visible. Run `node migration-work/footer-validation/scripts/validate-output.js <file.json> <references/...-schema.json>` (paths relative to workspace or skill bundle). Fix required fields. |
| `validate-output.js` fails with module not found | Ajv not installed | In `migration-work/footer-validation/scripts/`: run `npm install` (skill ships `package.json` + lockfile). |
| Gate blocks phase-2 | `detect-footer-sections.js` not run | Run script with `--url=`; confirm `.section-detection-complete`. |
| Gate blocks mobile structural summary | Mobile source detection missing | Run `detect-footer-mobile-sections.js` on **source** URL; confirm `.mobile-footer-structure-detection-complete`. |
| Hook blocks `footer.css` | No `footer-appearance-mapping.json` (source), or missing **`layoutSpacing`** / empty required spacing fields | Phase 2b: add `layoutSpacing` with `footerPaddingTop`, `footerPaddingBottom`, `contentInsetInline`, `columnGapApprox`, `majorBandGapApprox` (px strings from source `getComputedStyle`). |
| Hook blocks `footer-appearance-mapping.json` or desktop complete / Stop | Phase-1/phase-2 patterns require **`promoMediaBand`**, **`primaryLinkBand`**, or **`leadCaptureBand`** fields but mapping omits them | `footer-validation-gates/checks.js` (`checkMandatoryAppearanceParityBlocks`): tall image band → `promoMediaBand.present` + measured keys; large link nav → `primaryLinkBand`; `formType: inline-form` → `leadCaptureBand` with `fieldLayoutDesktop`. |
| Hook blocks `phase-4-mobile.json` or desktop “incomplete” | Source appearance exists but migrated mapping or register missing | Create `migrated-footer-appearance-mapping.json`, run `compare-footer-appearance.js` → `footer-appearance-register.json` with `allValidated: true` (same inputs as Step 6). |
| Orphan `migrated-footer-appearance-mapping.json` | Source mapping missing | Restore `footer-appearance-mapping.json` (Phase 2); then run compare. |
| Structural compare fails | Section **order** (by index), **type** per section, link/image/brand counts, form/locale/video flags | Desktop compare defaults to **100%** similarity **and** `allValidated` (strict per-section). Re-check phase-1 `heightMismatch`; ensure `migrated-structural-summary.json` lists sections **top-to-bottom** matching phase-2 **types**; fix `footer.plain.html` / `footer.js`; re-extract; re-run `compare-footer-structural-schema.js`. |
| Image **layout** wrong but counts match | Structural compare does **not** measure band height, `object-fit`, or column topology | Phase 2b: add **`promoMediaBand`** (height, width behavior, `objectFit`, order vs form) when a large image strip exists; **`primaryLinkBand`** when desktop nav is multi-column vs accordion; **`leadCaptureBand.fieldLayoutDesktop`** for single-row vs stacked fields. Re-extract migrated appearance and re-run `compare-footer-appearance.js`. Use side-by-side screenshots in Step 7. |
| Behavior register failures | Hover/click not tested on migrated page | Use **Playwright MCP** hover/click — not `element.click()`. Re-extract `migrated-footer-elements-mapping.json`; re-run compare. |
| Mobile behavior register failures | `migrated-mobile-behavior-mapping.json` vs `phase-4-mobile.json` | Align accordion mode, stacking length, form/locale/video strings; re-run `compare-footer-mobile-behavior.js`. |
| Image audit fails | Missing files or wrong paths | Download to `content/images/`; use relative `images/...` in `footer.plain.html`. |
| Pre-completion fails | Register not `allValidated` or ESLint on `footer.js` | Fix implementation; re-run compares; fix `blocks/footer/footer.js` lint (validation scripts are not lint-gated). |
| `detect-footer-sections.js` — no footer found | Site uses `div`/section with footer **class/id**, not `<footer>` / `role="contentinfo"` | Script retries after `networkidle` and matches generic `id`/`class` patterns (`footer`, `site-footer`, `page-footer`, etc.). If still empty, inspect DOM and ensure the footer container is visible after cookie dismissal. |
| Cookie wall blocks footer | CMP in another language / custom markup | Detection uses **structural** selectors first, then **one English** pattern (`Accept` / `Accept all`), then a **language-agnostic** pass (known CMP roots + reject “settings/reject” labels + last visible button). For odd banners, re-run with `--cookie-selector=<css>` on both `detect-footer-sections.js` and `detect-footer-mobile-sections.js`. |
| Footer fragment 404 / empty in preview | Default fragment path vs `content/footer.plain.html` | Ensure page metadata (or project defaults) point the footer fragment to the path where `footer.plain.html` is served; plain HTML should stay authoring-friendly — heavy decoration belongs in `footer.js`. |
| `footer.js` selectors miss after load | EDS **`decorateButtons()`** rewrites `<p><strong><a>` → `.button-wrapper` / `.button` (not `button-container`) | Query decorated DOM (`a.button`, `.button-wrapper a`, etc.); avoid relying on `<strong>` wrappers that authoring pipeline strips. |
| `footer.plain.html` wrong location | File at repo root | Must be `content/footer.plain.html`. |
| Hardcoded locale or copy in `footer.js` | Content-first violated | Move strings and image refs into `footer.plain.html`; hook warns on patterns in `footer.js`. |
| Missing content proceeded with a note | Register not used | Add `missing-content-register.json` items; implement; set `resolved: true`; re-run `validate-footer-content.js`. |
| Mobile-only content affects desktop | No `@media` isolation | Hide default desktop; show only in mobile breakpoint in `footer.css`. Track in `mobile/missing-content-register.json` if needed. |
| Script log missing in `debug.log` | Script not executed | Re-run script; read last 20 lines of `debug.log` for `[SCRIPT:...]`. |

## Nav vs footer (scope)

Navigation orchestrator includes megamenu behavior, row-elements validation, style registers, visual critique sub-agents, and `mobile-dimensional-gate.js` (full-width mobile menu layout). The footer orchestrator does **not** include those; mobile parity is enforced via **mobile structure detection**, **mobile structural compare**, **mobile behavior compare**, and **missing-content** registers.
